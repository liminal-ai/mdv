Authentication Flow Overview

This document describes the authentication and authorization flows used by the auth-service, including OAuth2 provider integration, token lifecycle, and session management.

Supported Flows

  • Authorization Code + PKCE — primary flow for web and mobile clients
  • Client Credentials — for service-to-service authentication
  • Refresh Token Rotation — automatic rotation with reuse detection

Token Lifecycle

Access tokens are short-lived JWTs (15 min TTL) signed with RS256. Refresh tokens are opaque, stored server-side, and rotated on every use. Reuse of a consumed refresh token triggers immediate revocation of the entire token family.

Token TypeFormatTTLStorage
Access TokenJWT (RS256)15 minClient only
Refresh TokenOpaque30 daysServer (Redis)
ID TokenJWT (RS256)1 hourClient only

Session Management

Sessions are tracked per-device using a fingerprint derived from the User-Agent and a client-generated device ID. The session store supports:

  • Concurrent session limits (configurable per tenant)
  • Remote session revocation via admin API
  • Idle timeout with sliding window renewal

See also: oauth2-provider-integration-guide-and-configuration-reference.md for provider-specific configuration, and token-rotation-strategy.md for the rotation algorithm details.
